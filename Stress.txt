Establish Stress Ground Truth (Creating Labels)

The first challenge is: How do we know whether a user is stressed?

AI-Based Mental Stress Detection from Typing Patterns and Smartphone Usage
The app data alone has no stress label. During research/development, a dataset is created by collecting:

A. Smartphone Behavior Data

Examples:

Typing speed
Typing pauses
Error rate
Screen time
App switching
Night phone usage
B. Stress Measurement Data (Labels)

Users periodically complete validated psychological questionnaires.

Common methods:

Perceived Stress Scale (PSS)

A widely used questionnaire where users answer questions about their stress level.

Example scoring:

PSS Score	Stress Category
0–13	Low stress
14–26	Moderate stress
27–40	High stress

The collected smartphone features are paired with the questionnaire result.

Example dataset:

Typing Speed	Errors	Screen Time	Night Usage	PSS Score	Label
45 WPM	5	4 hrs	20 min	10	Low
35 WPM	15	7 hrs	2 hrs	22	Moderate
28 WPM	25	9 hrs	3 hrs	32	High

This becomes the training dataset.

2. Feature Extraction Methodology

Raw app data is converted into measurable indicators.

Typing-Based Features
Typing Speed

Formula:

Typing Speed=
Time in minutes
Number of words
	​


Example:

200 words / 5 minutes

= 40 WPM

Possible stress indication:

Reduced speed
More pauses
More corrections
Typing Error Rate

Formula:

Error Rate=
Total Characters
Backspaces+Corrections
	​


Example:

20 corrections / 500 characters

= 4% error rate

Higher error rate may indicate distraction or reduced concentration.

Typing Rhythm Variation

The system calculates changes in typing intervals.

Example:

Normal:

120ms 125ms 118ms 122ms

High variation:

100ms 250ms 90ms 300ms

Feature:

Standard deviation of key intervals
3. Smartphone Behavior Indicators

The model calculates features such as:

Screen Usage Change

Example:

Normal week:

4 hours/day

Current week:

8 hours/day

Feature:

Usage Increase = 100%
Night Usage Ratio

Formula:

Night Usage=
Total Usage
Usage between 12AM−5AM
	​


Higher values may indicate sleep disturbance patterns.

App Switching Frequency

Example:

200 app switches/day

may indicate restless usage behavior.

4. Machine Learning Model Learns Patterns

After feature extraction:

Input:

[
Typing speed,
Error rate,
Pause duration,
Screen time,
Night usage,
App switching
]

Output:

Stress Category

The model learns relationships:

Example:

During training:

Low Stress Users

Typing:
- Faster
- Consistent rhythm
- Less correction

Phone:
- Lower screen time
- Normal sleep pattern


High Stress Users

Typing:
- Slower
- More pauses
- More errors

Phone:
- Increased late-night usage
- Frequent checking

The AI does not use one feature. It combines many signals.

7. Research Methodologies Used in Existing Studies

Many research systems use:

1. Supervised Learning

Most common approach.

Algorithms:

Random Forest
SVM
XGBoost
Neural Networks

The model learns from examples:

Behavior Data → Known Stress Level
2. Time-Series Analysis

Stress changes over time, so researchers analyze patterns:

Example:

Day 1: Normal
Day 2: More phone use
Day 3: Poor sleep
Day 4: High stress probability

Models:

LSTM
GRU
Temporal CNN
3. Personalized Baseline Method

Because everyone types differently, the system compares a user against their own normal behavior.

Example:

User's normal:

Typing speed = 50 WPM
Screen time = 4 hours

Current:

Typing speed = 32 WPM
Screen time = 9 hours

The system detects a behavioral change.
